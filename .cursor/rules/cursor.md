# 🚀 Cursor Regeln – FreelancerOS

## 🧠 1. Plan Mode zuerst

Bevor ich etwas baue oder einen Bug fixe:

* Ziel klar definieren
* In kleine Schritte aufteilen
* Erst denken, dann coden

Wenn etwas unklar ist oder nicht funktioniert → zurück in Plan Mode.

---

## 🧩 2. Komplexität aufteilen

Bei schwierigen Aufgaben:

* Problem in kleine Teile zerlegen
* Schritt für Schritt lösen
* Code einfach und sauber halten

Kein Chaos-Code.

---

## 🎯 3. Self-Improvement Loop (SEHR WICHTIG)

Jede Aufgabe soll mich besser machen.

Bei jedem Code:

* Was macht er?
* Warum ist er so geschrieben?
* Was passiert, wenn ich ihn ändere?
* Wo gehört er in die App?

Ziel: Verstehen, nicht nur fertig werden.

---

## ✍️ 4. Aktives Lernen (Tippen statt blind kopieren)

* Wichtige Logik tippe ich selbst
* Copy-Paste ist erlaubt bei:

  * CSS
  * wiederholendem Code
  * Boilerplate

Wenn ich unsicher bin → selbst tippen.

---

## 🧪 5. Verifizieren bevor ich es nutze

Bevor ich Code benutze:

* Logisch durchdenken
* Mögliche Fehler überlegen
* Prüfen ob es zum System passt

---

## 🐛 6. Bugs direkt lösen

Wenn ein Fehler auftaucht:

* Stoppen
* Ursache verstehen
* Sauber lösen (keine schnellen Hacks)

---

## 📚 7. Dokumentation (project.md)

Nach Fortschritt:

* Was habe ich gebaut?
* Was habe ich gelernt?

Einfach und klar halten.

---

## ⚖️ 8. Code Qualität > Geschwindigkeit

* Sauberer Code ist wichtiger als schneller Code
* Einfach > kompliziert
* Kein Overengineering

---

## 🚫 9. Kein Tool-Chaos

* Fokus auf Bauen, nicht Tools suchen
* AI ist Hilfe, nicht Ersatz
* Umsetzen > Perfektion

---

## 📈 10. Build → Test → Improve

Workflow:

1. Kleine Sache bauen
2. Testen
3. Verbessern

Wiederholen.

---

## 💰 11. Ziel nicht vergessen

Das Ziel ist NICHT:

* perfekter Code
* fancy AI Systeme

Das Ziel ist:
→ echte Nutzer
→ echtes Problem lösen
→ Schritt für Schritt besser werden

---

## 🔥 12. Daily Regel

Jeden Tag mindestens:

* 1 Fortschritt (Feature, Fix oder User-Kontakt)

Konstanz gewinnt.
